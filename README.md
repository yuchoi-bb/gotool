# gotool

즐겨찾기(바로가기) 런처. 실행하면 마우스 커서 위치에 우클릭 메뉴 모양의 **다열 패널**이 뜹니다.

```
⌨ 명령어   |   🌐 웹   |   💻 개발   |   📦 비개발   |   🐿 다람쥐      (기본 구성 — 설정으로 이름 변경/열 추가 가능)
```

- **왼쪽 클릭** → 해당 항목 실행(웹은 브라우저, 앱은 실행, 폴더는 탐색기)
- **드래그 & 드롭** → 다른 열로 이동(열간) 또는 같은 열 안에서 순서 변경(행간). 창은 닫히지 않고 제자리 유지
- **오른쪽 클릭** → 삭제(확인창)
- **`!` 표시(이름 오른쪽)** → 추가된 지 10분 이내의 신규 항목
- **⬆ 새 버전 업데이트** → 새 릴리스가 나오면 맨 위에 버튼이 자동 표시. **클릭 한 번**이면 직접 내려받아 exe를 자동 교체하고 새 버전으로 재실행 (작업 관리자·수동 복사 불필요)
- **ESC 또는 바깥 클릭** → 닫기(트레이 상주)

## 트레이 상주 + 더블 ESC (즉시 실행)

gotool은 한 번 실행하면 **트레이에 상주**합니다. 패널을 닫아도 프로세스는 살아 있어서 다시 열 때 프로세스 시작 비용(디스크 로딩, 백신 검사)이 전혀 없습니다.

- **ESC 두 번(0.4초 이내)** → 커서 위치에 패널 즉시 표시 (내장 기능 — AutoHotkey 불필요!)
- **트레이 아이콘 클릭** → 패널 표시
- **트레이 아이콘 우클릭** → `열기 / Windows 시작 시 자동 실행(추천) / 종료`
- `gotool.exe`를 다시 실행해도 새 창을 띄우지 않고 상주 인스턴스가 패널을 보여줍니다
- 더블 ESC를 끄려면 `config.json`에 `"disableEscHotkey": true` 추가

**빠른 설정**: 트레이 우클릭 → "Windows 시작 시 자동 실행" 체크. 이후에는 부팅하면 자동으로 대기하고 있다가 ESC 두 번에 즉시 나타납니다.

## 자동 분류

`shortcuts` 폴더에 섞어 넣어도 자동으로 분류됩니다.

| 항목 | 분류(auto) |
|---|---|
| `.url` (http/https 주소) | `web` |
| 이름·대상 경로에 개발 키워드(code, git, docker, python, sql, terminal 등) | `dev` |
| 그 외 앱/파일/폴더 | `etc` |
| auto 미지정 열(기본: 🐿 다람쥐) | 수동 이동 전용 |

분류가 마음에 안 들면 **항목을 드래그해서 원하는 열/순서에 놓으면** 됩니다. 순서는 `.order` 파일에 저장되어 유지됩니다.

## ⌨ 명령어 열 (CMD 실행)

맨 왼쪽 **명령어 열**은 자주 쓰는 CMD 명령을 모아두는 곳입니다.

- 열 맨 아래 **"＋ 명령어 추가"** 클릭 → `실행할 명령어`(예: `aws sso login`)와 `표시 이름`(선택) 입력
  - 표시 이름을 비우면 명령어가 그대로 항목 이름으로 보입니다 (보이는 것과 실행되는 것을 다르게 할 수 있음)
- 항목 **클릭 → cmd 창이 열리고 명령이 자동 실행**됩니다. 명령이 끝나도 **cmd 창은 닫히지 않고 유지**됩니다 (`cmd /k`)
- 여러 명령 연결: `명령1 && 명령2`, 또는 저장된 `.cmd` 파일을 메모장으로 열어 여러 줄로 편집
- 내부적으로 `shortcuts\명령어\` 폴더에 `.cmd` 파일로 저장되므로 드래그 이동·우클릭 삭제·순서 변경·백업이 모두 동일하게 작동합니다

## 설정 (열 이름 수정 · 열 추가)

패널 하단 **"⚙ 설정"** 버튼을 누르면 **설정 창**이 열립니다.

- **열 이름**: 패널에 표시되는 제목을 입력창에서 바로 수정
- **폴더 이름**: 그 열의 항목이 저장되는 `shortcuts` 하위 폴더 이름
- **자동 분류**: 버튼을 눌러 `웹주소 / 개발 / 기타 / 없음` 순환 선택 (없음 = 수동 전용 열)
- **＋ 열 추가** / 행 끝의 **✕** 로 열 추가·삭제 (최소 1개)
- **저장**(Enter) 하면 즉시 패널에 반영, **취소**(ESC)로 닫기

설정은 `shortcuts\config.json`에 저장되므로 백업에 포함되며, 파일을 직접 편집해도 됩니다.
(`devKeywords` 배열을 추가하면 "개발" 자동 분류 키워드를 직접 지정할 수 있습니다)

## 백업 · 복원 (포맷 대비)

모든 데이터(바로가기, 열 구성 `config.json`, 순서 `.order`, 신규 기록 `.seen`)는 `shortcuts` 폴더 안에 있습니다.

- **💾 백업 저장**: 패널 하단 버튼 → 저장 위치 선택 → `shortcuts` 폴더 전체가 zip 하나로 저장
- **📂 백업 불러오기**: 포맷/재설치 후 gotool 실행 → 버튼 클릭 → zip 선택 → 그대로 복원

## 바로가기 추가

1. **탐색기 우클릭(추천)** — 한 번만 `gotool.exe install` 실행(또는 패널 하단 등록 버튼)하면
   파일/폴더 우클릭 메뉴에 **"gotool에 추가"**가 생깁니다. (Windows 11은 "더 많은 옵션 표시" 안)
   - 웹: 브라우저 주소창의 자물쇠를 바탕화면에 드래그해 `.url`을 만든 뒤 우클릭 → "gotool에 추가"
2. **폴더에 직접 넣기** — `.lnk`/`.url`/파일을 `shortcuts` 폴더(또는 열 하위 폴더)에 복사
3. **명령행** — `gotool.exe add "C:\경로"`

## 수정 · 삭제

- **삭제**: 항목 우클릭 → 확인
- **수정(이름 변경 등)**: 패널 하단 **"📁 바로가기 폴더 열기"** → 파일 이름을 바꾸면 표시 이름이 바뀝니다
- 우클릭 메뉴 등록 해제: `gotool.exe uninstall`

## 오류 로그 (디버깅)

오류가 나면 exe 옆 **`gotool.log`**에 자동 기록됩니다. 형식(탭 구분):

```
2026-07-13 15:04:05    v0.5.0    E03    이동 실패: ...
```

| 코드 | 유형 |
|---|---|
| E01 | 폴더 생성/접근 실패 |
| E02 | 창 생성 실패 |
| E03 | 항목 이동 실패 |
| E04 | 항목 삭제 실패 |
| E05 | 항목 추가 실패 |
| E06 | 우클릭 메뉴 등록/해제 실패 |
| E07 | 업데이트 확인 실패(기록만) |
| E08 | 백업/복원 실패 |
| E09 | 설정 파일(config.json) 오류 |
| E10 | 자동 업데이트(exe 교체) 실패 |
| P01 | 느린 시작 프로파일(내부 처리 300ms 초과 시 기록) |
| E99 | 예기치 못한 오류(패닉) |

오류창에도 `[E03]`처럼 코드가 함께 표시됩니다. 문제가 생기면 `gotool.log` 내용을 알려주세요.

## 실행이 느릴 때

gotool은 폴더 목록만 읽고 즉시 패널을 띄우며, 아이콘·분류 결과는 캐시(`shortcuts\.iconcache`, `.meta`)로 재사용합니다.
그런데도 느리다면 `gotool.log`의 `P01` 기록을 확인하세요.

- **P01이 없거나 수십 ms인데 느리다** → 앱 바깥 문제입니다. 대부분 Windows Defender가 서명 없는 exe를
  실행할 때마다 검사하는 것이 원인이며, `Windows 보안 → 바이러스 및 위협 방지 → 제외 추가`에
  gotool 폴더를 등록하면 해결됩니다.
- **P01 수치가 크다** → 그 내용을 이슈로 알려주세요.

## AutoHotkey 연동 (선택)

**더블 ESC가 내장되어 있어 AutoHotkey는 더 이상 필요 없습니다.** 기존 AHK 스크립트를 그대로 둬도
동작은 하지만(상주 인스턴스에 표시 신호만 전달), 내장 단축키를 쓰는 편이 훨씬 빠릅니다.
다른 키를 쓰고 싶을 때만 AHK로 `Run "C:\tools\gotool.exe"`를 연결하세요.

## 명령어 정리

| 명령 | 동작 |
|---|---|
| `gotool.exe` | 패널 표시 (exe 옆 `shortcuts` 폴더) |
| `gotool.exe <폴더>` | 지정 폴더의 내용을 패널로 표시 |
| `gotool.exe add <경로>` | 바로가기 추가 (`.lnk`/`.url`은 복사, 그 외는 바로가기 생성) |
| `gotool.exe install` | 탐색기 우클릭 "gotool에 추가" 등록 |
| `gotool.exe uninstall` | 우클릭 메뉴 등록 해제 |

## 다운로드 / 빌드

[Releases](../../releases/latest) 페이지에서 `gotool.exe`를 내려받으세요.

```
go build -trimpath -ldflags "-s -w -H windowsgui -X main.version=v0.14.0" -o gotool.exe .
```

`main`에 푸시하거나 `v*` 태그를 만들면 GitHub Actions가 자동으로 빌드해 릴리스에 `gotool.exe`를 올립니다.
